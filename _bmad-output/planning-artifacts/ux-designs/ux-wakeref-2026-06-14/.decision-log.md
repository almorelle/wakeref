Run: ux-wakeref-2026-06-14 · Scope: **Home page only (feature launchpad)**
Canonical record of decisions for the home redesign.

> **Note (2026-06-14):** the design-handoff scaffolding was pruned at commit — the HTML mocks (`.working/`), the Stitch handoff prompt, and the empty DESIGN.md / EXPERIENCE.md spine stubs were deleted once the redesign shipped in code. ## Inherited facts (brownfield — not decisions, ground truth)

- **Existing design system** lives in `src/index.css`: dark-first with `[data-theme="light"]`, tokens for colors / radius / transitions, Syne (display) + DM Sans (body) self-hosted.
  - Core colors: bg `#0a0a0f`, surface `#111118`/`#1a1a24`/`#22222f`, text `#f0f0f8`, muted `#8888a0`, accent `#3b82f6` / accent2 `#60a5fa`, wake `#06b6d4`, wakeskate `#f59e0b`, submit/violet `#7c3aed`.
  - Radius: sm 6 / md 10 / lg 16 / xl 24. Brand mark: `/picto.svg` (helmet) as CSS mask (`.picto-mark`).
  - Global components already exist: `.btn` (primary/ghost/danger/submit/icon), `.card`, `.badge-*`, `.section-title`, `.spinner`, `.toast`. Home redesign **reuses these**, does not reinvent.
- **Foundation = the existing CSS Modules + global tokens UI system.** DESIGN.md documents/extends it; EXPERIENCE.md specifies only the behavioral delta for the home.
- **Form factor:** mobile-first PWA. Usage context (from brainstorming): wet hands, full sun, on a phone, between two runs. Also serves desktop.
- **i18n:** FR (default) + EN; strings in `src/i18n/translations.js`. No-auth public.

## Established direction (from brainstorming 2026-06-13, idea #23)

- **Problem:** discoverability — riders find the Tricks page but do NOT realize Quiz and Compo exist. Best features are hidden.
- **Direction (validated):** home becomes a **feature launchpad** (big per-feature entry points). No need to list all 14 categories on the home (the Tricks page already does that).
- **The 6 invariants filter every choice:** no-auth by default · survive abandonment · anti-ego/anti-perf (include, never rank) · self-service best-effort · inclusion by *filling* not labeling · grow by curation not production.
- **Representation (#24):** the home is a representation surface — default display/order is curated, men shown first but the sport shown as for everyone; *felt, never labeled* (no gendered badges).

## Open decisions (to elicit)

- D2 — Keep / drop from current home (search, submit CTA+stats, categories grid, WCT banner, recent figures, recent videos)
- D3 — The single-use moment: what a rider leaves the home with

## Decisions

### D1 — Launchpad tiles: Tricks · Quiz · Compo (3) — 2026-06-14
Three feature tiles: **Tricks catalogue**, **Quiz**, **Compo/run**. Parcours/collections NOT included (not built yet; revisit when shipped). Directly fixes #23 by giving Quiz and Compo first-class home presence.

### D2 — Hierarchy: search hero + tiles below — 2026-06-14
Keep the **search bar as the hero centerpiece** (preserves today's #1 usage), with the 3 feature tiles immediately below. Continuity + discoverability compromise (not a pure tile wall).

### D3 — Strip the home to search + tiles only — 2026-06-14
Keep **only the search bar** from the current home. Removed: stats counter + submit-video CTA, recent figures, recent videos, World Cable Tricks banner, and the 14-category grid (per #23). Home becomes radically simple.
**[RESOLVED 2026-06-14]** Wushu confirms NO representation/curation trace on the home. Parity (#24) is felt inside the features themselves; video upload (#6) already has entry links on the trick (figure detail) pages. The strip stands — home = search + 3 tiles only.

### D4 — Single-use moment: choose a feature — 2026-06-14
The home is a **crossroads**: in ~5s a rider grasps the 3 things WakeRef does and picks one. Pure launchpad intent. Coherent with D3's stripping.

### D5 — Handoff prompt assembled (Google Stitch) — 2026-06-14
`stitch-handoff-prompt.md` written: self-contained producer prompt encoding the inherited brand tokens (Syne/DM Sans, dark+light palette, helmet mark, radius scale), the home layout (hero+search centerpiece → 3 equal tiles → nothing else), per-tile draft microcopy FR/EN, the 3 states (default / search-active / no-results), and the hard anti-ego/no-auth/a11y rules. Routes & icons for build: Catalogue `/figures` (list) · Quiz `/quiz` (help) · Compo `/compo` (calculator).
**Draft microcopy** is the author's to tweak before pasting. EXPERIENCE.md to be distilled in **Update** mode once Stitch outputs land in the run folder.

### D6 — Hero title kept as-is — 2026-06-14
Keep "Tous les tricks, une seule référence." / "Every trick, one reference." for now; broaden later if needed based on how the launchpad reads. No prompt change.

### D7 — Stitch handoff abandoned; switched to on-brand HTML mock — 2026-06-14
Stitch output rejected (ugly, reinvented the identity, even generated an unwanted new logo). For a brownfield app with a strong existing system, generation-from-scratch is the wrong tool. Built faithful HTML mocks from the real tokens/components instead: `.working/home-launchpad-mock.html` (minimal) + `.working/home-rich-mock.html` (rich). The existing helmet `picto.svg` stays — never regenerated.

### D8 — SUPERSEDES D3: keep the rich home, swap category grid → feature tiles — 2026-06-14
The minimal/stripped home felt too empty. Final layout: hero (search) → **3 feature tiles** (Catalogue/Quiz/Compo) placed directly under the hero, **replacing** the old categories+contexts grid → contribution CTA + stats → World Cable Tricks banner → recent figures → recent videos. This solves #23 (Quiz/Compo now visible) AND keeps representation (#24, via recents) + the curation loop (#6, via the submit CTA). Categories now reachable only via the Tricks page (consistent with #23). Tile accents: Catalogue=accent blue, Quiz=wake cyan, Compo=wakeskate amber.
